SEZE — Developer App Report

Full-stack vetting, remediation and backend verification of the SEZE women's fitness & nutrition app. Covers a 7-persona audit, security hardening, real data persistence, and a prioritised risk register.
Date 19 Jul 2026 Version 1.0.0-beta Live rosa-app-ten.vercel.app Repo github.com/JosephMichaelDaniels/Rosa Backend Supabase (cttyprfieqydaiifncbu)

01 — Executive summaryWhat changed

This engagement moved SEZE from a visually-polished shell with a broken backend to a functional, secured app where real accounts persist and personalise correctly. A seven-expert audit (product, design, front-end, back-end, nutrition, QA, security) surfaced 120 raw findings, consolidated to 70. The dominant defect was systemic: the app read/wrote a profiles table that does not exist (real table: rosa_users), so onboarding, login routing, personalised targets, workouts and logging all silently failed. That class of bug is now fixed and verified end-to-end against the live database. Alongside, three security holes (a nutrition-data RLS leak, feedback spoofing, mutable function search_path) and two clinical-safety gaps (calorie deficits for pregnant/breastfeeding users) were closed.

03 — Security & safety fixesCritical closures

SEC-01 · Data leak (critical)

The rosa_daily_nutrition view ran as owner (default security_invoker=off), bypassing RLS — any anonymous caller could read every user's nutrition totals. Fixed: security_invoker=on + revoked anon access.

NU-02 · Clinical safety (critical)

Fat-loss deficits could be prescribed to pregnant / breastfeeding users. Fixed: life-stage now overrides goal — perinatal stages get energy added, never subtracted, with hard kcal floors.

SEC-03 · Feedback spoofing

Insert policy was WITH CHECK (true) — anyone could write feedback as any user. Fixed: authenticated-only, auth.uid()=user_id.

SEC-04 · Function hardening

SECURITY DEFINER functions had mutable search_path. Fixed: pinned to empty; revoked needless public RPC on the signup trigger.

07 — Recommended roadmapNext, in priority order

  • Programme content engine — real per-programme exercise libraries with form video, and week/day progression (replaces the single demo session). Highest-value gap.
  • Perinatal safety gate — medical-clearance confirmation before heavy loading for pregnancy / early-postnatal; physician-advised flag.
  • Nutrition depth — wire the search UI to the new function, log fibre/sugar/sodium, add barcode scan; obtain USDA key.
  • Quick-track & progress — persist Water/Mood/Weight (table ready) and drive Progress charts + badge-earning from real sessions.
  • Account controls — edit profile, change goal/programme, notifications, set-new-password screen for recovery.
  • Brand-value features — progress photos, cycle awareness, pelvic-floor guidance, sleep/recovery, community — expected by the postnatal audience.
  • Hardening for scale — enable leaked-password protection + CAPTCHA, server-side consent records, branded SMTP emails, precompiled build + service worker.
